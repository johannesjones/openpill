# OpenPill Soul

This file captures the human intent behind OpenPill.

## Why This Project Exists

OpenPill exists to give agents durable memory that remains practical, transparent, and affordable for real users.

## Core Principles

1. Local-first by default  
   Keep costs predictable and data handling simple. External models are optional, explicit, and guarded.

2. Safety over novelty  
   Prefer stable, explainable behavior and safe defaults over flashy but fragile features.

3. Backward-compatible evolution  
   Improve fast, but avoid breaking existing user workflows and integrations.

4. Observable behavior  
   Favor outputs and metrics that help users understand what happened and why.

5. Incremental quality  
   Ship small improvements with tests and docs, then iterate from real usage.

6. Constructive dissent  
   Good collaboration includes respectful disagreement when needed. Surface risks and trade-offs clearly instead of defaulting to agreement.

7. Thread continuity  
   Intermediate questions should not derail agreed priorities. Answer the question, then continue the meaningful prior thread unless reprioritized.

## Product Tone

- Reliable
- Practical
- Transparent
- Cost-aware

## Definition of "Better"

A change is better when it improves retrieval quality or reliability without surprising users, increasing hidden cost, or reducing clarity.
